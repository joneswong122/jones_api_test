The following article provides a reference to all available Connatix Player JavaScript API calls and events.

Scope

There are two classifications for the Connatix Player, which will be referenced in this documentation as Instream and Outstream. The Instream player plays video content and ads, while the Outstream player only plays ads.

The functions and settings mentioned below can be applied either to only one type of player or both types of players at the same time.

The functions will be referenced in the documentation as Scope. For example, <<scope: Instream/Outstream>> will apply to both types of player, whereas <<scope: Instream>> will apply only to the Instream player.

Setup

Before you start configuring the behavior of your embedded Connatix Player, you'll first need to integrate it inside your page and test that it renders properly.

To load the player inside your page, you have to add and configure the required scripts correctly.

Load the player

Place this code snippet in the  head  section of your page:  

<script>
! function(n) {
if (!window.cnx) {
window.cnx = {}, window.cnx.cmd = [];
var t = n.createElement("iframe");
t.display = "none", t.onload = function() {
var n = t.contentWindow.document,
c = n.createElement("script");
c.src = "//cd.connatix.com/connatix.player.js", c.setAttribute("async", "1"), c.setAttribute("type", "text/javascript"), n.body.appendChild(c)
}, n.head.appendChild(t)
}
}(document);
</script>

 Initialize, render and control the player

Place this code snippet in the  body  section of your page: 

<script id="{{scriptId}}">
cnx.cmd.push(function() {
cnx({
playerId: "{{playerId}}"
}).render("{{scriptId}}", (renderError, playerApi) => {
// player is ready and rendered
});
}
</script>

Let's analyze how the  body  script works.

  • The player script on the page window defines the  cnx.cmd  array. When the player is loaded, it will call all functions pushed into this array. If the player script has already been loaded, pushing to this array will call the function immediately.

  • After the player is initialized with its settings, we have to render it to get the player API and events. To render the player, we call the render function with the id of the script tag where the call happens, or another id of an element in the page. This element is provided as reference, and the player without unique settings will render above the component.

  • The render function can take a second parameter, which is a callback function that gets called when the player is rendered and ready and will be called with an API object parameter and an error object parameter.

Configuration

After you've tested that the player renders inside your page, you can go ahead and configure the player.

Replace the code snippet in the  body  section with the one you configure based on the instructions below. The  cnx(config)  function initializes the player with a configuration JSON. 

<script id="renderID">
cnx.cmd.push(function() {
cnx({
config
}).render('renderID');
});
}
</script>

The following JSON object is an example containing all necessary configuration options for initializing the Connatix Player. 

Use these two helpers to change attributes in an easy and type-safe way:

  • configEnums - helper enums for setting object attributes
  • configEvents - helper object containing all available events

Both helpers can be found on the  cnx  object (for example,  cnx.configEnums ).

After you configure the JSON object with your preferred settings, add it to the  body  script instead of  config .

Player Renderer

cnx.render(scriptId, readyCallback)

scope: Instream/Outstream

This function renders the player on the page, based on the scriptId given, and then calls the readyCallback (check its description in the Params table) after the player has rendered. 

<script id="{{scriptId}}">
cnx.cmd.push(function() {
cnx({
playerId: "{{playerId}}"
}).render("{{scriptId}}", (renderError, playerApi) => {
if (renderError) {
// An error occured while rendering the player
if (renderError.type === cnx.configEnums.ErrorTypesEnum.NoPlayer) {
// Player was not served or id given is incorrect
} else {
// There was an error rendering the player
}
return;
} else {
// Here the playerApi is safe to use
playerApi.play();
}
});
}
</script>

Params

scriptId An id of the script element (or any element in the page) which is used for rendering the player. The player will render above it if no special rendering settings are given at init (or in the management dashboard). Must be an id of an element in the rendered DOM of the page. Mandatory String
readycallback A callback function which will be called when the player is rendered in the page and initialized.   Optional Function

API Events

The following deals with the enumeration of events fired through the lifecycle of the player. All events can be subscribed to using the configEvents helper object described above (configEvents table). For example: 

// subscribe to play event using string
playerApi.on("play", function() {
// do something
});
// subscribe to play event using configEvents helper
playerApi.on(cnx.configEvents.Play, function() {
// do something
});

How to

Register with an anonymous function: 

playerApi.on("eventName", function(eventData) {
// do something
});

Register with a named function: 

playerApi.on("eventname", callbackFunction);
function callbackFunction(eventData) {
// do something
}

Fire event once: 

playerApi.once("eventname", callbackFunction);
// or with an anonymous function
playerApi.once("eventname", function(eventData) {
// do something
});

Remove event listeners: 

// Remove specific event listener with a named function
playerApi.remove("eventName", callbackFunction);
// Remove all event listeners
playerApi.removeAll();
// remove event listener using an anonymous function
// simply save the return function from the event subscription and then call that function to destroy it
var eventDestroy = playerApi.on("eventName", function(eventData) {
// do something
});
eventDestroy();

API Functions

The following is the player API object, which the callback function will be called with if the player has rendered successfully. Its methods are described below:

Meta

.getPlayerType()

scope: Instream

This returns the PlayerTypeEnum value which represents either the Instream or the Outstream player.

.getPlaylistId()

scope: Instream

This returns the id of the playlist currently playing.

.getVideoDetails()

scope: Instream

This returns an object containing the details of the video currently playing:

  • id: String.
  • title: String.
  • description: String.
  • duration: Number with seconds.
  • keywords: Array of strings.
  • customFields: array of custom metadata depicted as name - value  objects

If the video doesn't start ("videoStarted" event), then it will return undefined.

.getVideoCurrentPosition()

scope: Instream

This returns the position of the video currently playing in seconds.

.getVideoDuration()

scope: Instream

This returns the total duration of the video currently playing in seconds.

Playback

.play()

scope: scope: Instream/Outstream

For the Outstream player, it will play only if an ad that was running has been paused, otherwise this function does not apply.

This sets the player state to “play” 

playerApi.play();

.pause()

scope: scope: Instream/Outstream

For the Outstream player, it will pause only if an ad is playing.

This sets the player state to “pause” 

playerApi.pause();

Viewability

.isViewable()

scope: scope: Instream/Outstream

This returns a boolean which is true if the player is 50% in view and in an active tab.

It will be false if it’s below 50% or in an inactive tab. 

var isViewable = playerApi.isViewable();

.setViewableThreshold(threshold)

scope: scope: Instream/Outstream

This sets the viewable threshold, which means how much is the player in view. The threshold should be a numeric value between 0 and 1.

For example, this will set the viewable threshold to 0.5, meaning the player is 50% into view: 

playerApi.setViewableThreshold(0.5);

This will make the player completely out of view: 

playerApi.setViewableThreshold(0);

playerApi.setVolume(0.3);

let volume: number = playerApi.getVolume();

Destroy

.destroy()

scope: Instream/Outstream

This destroys the player, including the DOM rendered elements. After calling destroy, the last event the player sends is the "removed" event. By doing so, all functionality and events stop working on the API object.
